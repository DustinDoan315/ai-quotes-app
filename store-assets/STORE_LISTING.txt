Inkly: Daily Vibes — store upload checklist

Files in this folder (generated by: npm run icons:sync)
- app-store-icon-1024.png — Apple App Store (1024×1024), copied from assets/images/icon.png
- google-play-icon-512.png — Google Play high-res icon (512×512), if store-assets/playstore.png exists when you run icons:sync

App Store Connect (iOS)
1. Open App Store Connect → My Apps → your app → App Information.
2. Upload the app icon slot using app-store-icon-1024.png (or assets/images/icon.png after a 1024×1024 export).
3. Submit a new build from EAS (production profile) after icons are synced in the repo.

Google Play Console (Android)
1. Open Play Console → your app → Grow → Store presence → Main store listing.
2. Set the app icon using google-play-icon-512.png (512×512).
3. Feature graphic, screenshots, and short description are separate — use Google’s current size requirements.

Native rebuild (required for device icon changes)
1. npm run icons:sync
2. npm run build:production   (or: eas build --platform ios --profile production, etc.)
3. OTA / JS updates do not change the installed app icon.

Adaptive icon (Android)
If the logo looks cropped on the home screen, export a foreground PNG with safe-zone padding from your design tool and replace assets/images/android-icon-foreground.png, then rebuild.

Optional monochrome (Android 13+ themed icon)
Place store-assets/android-icon-monochrome.png and run npm run icons:sync — it will copy into assets/images/android-icon-monochrome.png. If omitted, the existing file in assets/images is kept.

Optional Google Play source
Add store-assets/playstore.png (512×512) before npm run icons:sync to refresh google-play-icon-512.png. If omitted, an existing google-play-icon-512.png in this folder is left unchanged.
